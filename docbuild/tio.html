Terminal Input / Output (tio)¶

The Terminal input/output module provides a set of simple to call command line functions that read the and write to the terminal with defaults, range checking and error correction. There is also a simply option selector and escape commands that allows simple interactive applications to be written with the miminium of effort. This modulue is used for all terminal inout in the examples and test code and is also called for error recovery when wrong lens files or materials are specified.

Reading Basic Types¶

The simplest five function read the basic Python variable types; each prompt the used with an optional default, reads from the terminal and returns the specified type after optional sanity checking. These functions are:

tio.getString(prompt, default=None)[source]¶

Read a string from the terminal with no processing or evaluation.

Parameters

  • prompt (str) – The prompt to be displayed.

  • default (str) – the default string (defaults to None)

Returns

str with no processing, but leading and trailing white space will be removed.

Note: will allow a zero length string to be returned.

tio.getFloat(prompt, default=None, min=None, max=None)[source]¶

Read a float from the terminal with optional default and range checking.

Parameters

  • prompt (str) – the prompt to be displayed.

  • default (float) – float (defaults to None)

  • min (float) – float min value accepted (defaults to None)

  • max (float) – float max value accepted (defaults to None)

Returns

float in specified range.

Note: if response is a string it will be evaluated.

tio.getInt(prompt, default=None, min=None, max=None)[source]¶

Read an int from the terminal with optional default and range checking. The default is decimal but also binary (prefix 0b) , oct (prefix 0o) , hex prefix (0x) is also supported.

Parameters

  • prompt (str) – the prompt string to be displayed.

  • default (int) – default value, (defaults to None)

  • min (int) – min value accepted (defaults to None)

  • max (int) – max value accepted (defaults to None)

Returns

int in specified range.

Note: if response is a string it will be evaluated.

tio.getBool(prompt, default=None)[source]¶

Read a boolean from the terminal with checking. It will accept: yes / no , true / false in lower or upper case, 1 / 0 or any logical expression.

Parameters

  • prompt (str) – the prompt to be displayed.

  • default (bool) – default response (default to None)

Returns

bool True/False

Note: if response is a string it will be evaluated.

tio.getComplex(prompt, default=None, maxabs=None)[source]¶

Read a complex from the terminal with optional default and range checking of the abs.

Parameters

  • prompt (str) – the prompt to be displayed.

  • default (complex) – the default (may be None)

  • maxabs (float) – maximum abs value of complex accepted (defaults to None)

Returns

complex in specified range (it will always return complex even if imaginary is zero)

It will also try and convert from list or tuple, and evaluate strings.

Reading Vectors and Angles¶

The following three function real in the specific vector and angle classes used in the optics package, again with prompt, defaults and error checking.

tio.getVector2d(prompt, default=None, maxabs=None)[source]¶

Read a Vector2d from the terminal with default and checking for the abs maximum.

Format from terminal may be ‘x,y’ OR ‘[x,y]’, also each componet will be evaluated.

Parameters

  • prompt (str) – the prompt to be displayed

  • default (Vector2d) – the default value (may be None)

  • maxabs (float) – maximum absolutle value of the Vector2d, (defaults to None)

Returns

a Vector2d in specified range.

Note: strings will be evaluated to try and form a Vector2d.

tio.getVector3d(prompt, default=None, maxabs=None)[source]¶

Read a Vector3d from the terminal with checking.

Format from terminal may be ‘x,y,z’ OR ‘[x,y,z]’, also each componet will be evaluated.

Parameters

  • prompt (str) – the prompt to be displayed

  • default (Vector3d) – the default value (may be None)

  • maxabs (float) – abs max value of the Vector3d (defaults to None)

Returns

the set Vector3d

Its also Tries to evaluate any sensible string as a Vector3d.

tio.getAngle(prompt, default=None)[source]¶

Read a Angle in theta/psi fromat the terminal with checking.

Format from terminal may be ‘theta,psi’ OR ‘[theta,psi]’, also each componet will be evaluated.

Parameters

  • prompt (str) – the prompt to be displayed

  • default (Angle) – the default value (may be None)

Returns

the Angle (note will always return an Angle)

Note: input values are in radians.

tio.getAngleDegrees(prompt, default=None)[source]¶

Read a Angle in theta/psi from the terminal in Degrees with checking.

Format from terminal may be theta,psi OR ‘[theta,psi]’, also each componet will be evaluated.

Parameters

  • prompt (str) – the prompt to be displayed

  • default (Angle) – the default value (may be None), this is assumes to be in radians.

Returns

the Angle (note will always return an Angle in radians)

Note: input values are in degrees but the returned Angle values are in radians.

File and Filenames¶

The simplest and most commonly used file function prompts and a file, open it and returns a file descriptor, this being

tio.openFile(prompt, key='r', defaulttype=None, defaultname=None)[source]¶

Method to open a text file with sanity checking, optional defaults and reprompt on failure. This is the main used callable function to open files.

Parameters

  • prompt (str) – the prompt to be displayed

  • key (str) – the key passed to open, default is “r” (read)

  • defaulttype – the default extension which will be added if not supplied, (default to None)

  • defaultname (str) – the defaault filename, (defaults to None)

Returns

the the opened file descriptor.

The file names is processded to expand environmental variable and user names so for example $ENV/dir/file.data or ~user/dir/file.data are expanded

Finer control can be optained from

tio.getFilename(prompt, defaulttype=None, defaultname=None)[source]¶

Method to get a filename with optional defaults. The name is also processed by getExpandedFilename to process environmental variable with $env or ~username prefix to a filename.

Parameters

  • prompt (str) – the prompt to be displayed

  • defaulttype – the default extension which will be added if not supplied, (default to None)

  • defaultname (str) – the defaault filename, (defaults to None)

Returns

the filename as a str which has been expanded to deal with logical names and username prefix.

Which will prompt for the filename and return the string, but not open the file.

There is also an useful utility function that takes a string comtaining logical variables or user directives and expand to a full file name.

tio.getExpandedFilename(name)[source]¶

Method to expand a (Unix) filename and process environmental variable with $env or ~username prefix to a filename.

Parameters

  • name – with original name, assumed to contains NO leading white spaces

  • type – str

Returns

the expanded filename as a str.

Note this method does not prompt for input from the terminal.

Typical input is $HOME/data.data or ~fred/data.dat, where $HOME is env name and fred is username. This works under Linix and MacOS, untested on other OS.

Option and Menu¶

There is simple options / menu function that prompts for a option, and return the option number and opt ion as a truple. This allows simple interactive programs to be easily built.

tio.getOption(prompt, options, default=None)[source]¶

Method to get a choice of options from a supplied list with error checking.

Parameters

  • prompt (str) – the prompt to be displayed.

  • options (list) – the list of options assumed to be list of str.

  • default (int) – the default option (int in range 0 -> < len(options)) (default to None)

Returns

truple selected option, opt, so (opt,options[opt])

Each option is tested for existance and uniquness. It will fail an re-prompt as required. There is also a simple internal ‘help’ option that pints the list of options.

Print Function¶

There is a simple print function that replaces to standard print(), this is independant of Python 2/3 problems and also journal to a file if this option is set

tio.tprint(*args)[source]¶

Simply alternative to print that will print to the sysout and also to journal if there is a journal file open. Output to the journal file be prefixed with a comment character.

Parameters

args – argumemnt list each will be conveterd to a str() and concatinated to a single string.

Also newline will be appended if not present and the print buffer will be flushed.

Note this should work identially in Pythons 2 and 3 since it uses direct write to output rather than print()

Journal Function¶

There is a simple journal function that records all inpouts and outputs, inclduing what is prinnted by tio.tprint() to a journal file. This is under development and at the moment only output, it is planned to allows this journal file to be reused as input.

tio.setJournal(filename=None)[source]¶

Method of open / close a journal file that records prompts and commands typed to a text file. param filename string the name of the journal file, if None will close any current open journal file.

Parameters

filename (str) – the name of the journal file, (defaults to None which switches off journal)

This is elementary fucntion at the moment and may be expanded in scope.

Escape Commands¶

There is a small set of commands that work with all imput called. There are all refixed by %-sign. The command will be executed and then the it will re-prompt. The current set are, (more will added in future releases).

Command

Action

%beep

Rings the terminal bell

%exit

Exits the program

%journal fn

Open journal file fn

%nojournal

Switch off journal

%dir root

director or root

Examples¶

Simple example to read string, float and int and output the values via tprint()

import tio

s = tio.getString("Type a string")
x = tio.getFloat("And an float",3.5,1.0,10.0)
i = tio.getInt("Then an int",0,-10,10)
tio.tprint("Values typed are ",s,x,i)

Read a complex , with default and max absolute, and a logical with a if:else structure.

import tio

z = tio.getComplex("Complex number",complex(2,3),50.)
if tio.getBool("Boolean",True):
  tio.tprint("Typed True")
else:
  tio.tprint("Typed False")

Open a file for write access with a defaut type and default with logical default file name, this will expand the logical name $Home and also set a default extension of data which will be appended if not given.

import tio
file = tio.openFile("File","w","data","$Home/output")
file.write("Hello World\n")

Simple use of the options so select one of 4 options number 1 (“go”) as the default.

import tio

options = ["stop","go","reset","reformat"]
i,opt = tio.getOption("Which option",options,1)
tio.tprint("Option chosen was : ",i," being ",opt)
